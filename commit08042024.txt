Refined parser and utility files for cub3D project

Este commit implementa várias melhorias no processamento de linhas de
 configuração e do mapa, além de adicionar um arquivo log.c para ajudar na
 análise de problemas durante o desenvolvimento do projeto.

Alterações principais:

- Aprimoramento dos arquivos de parser, garantindo uma análise eficiente das 
linhas de configuração e de mapas de diversos tamanhos.

- Cria e refina arquivos dedicados a validações, tratamento de erros e 
manipulação de strings.

Sigo enfrentando problemas para a configuração do projeto cub3D no WSL2, dando
 problemas na interação efetiva com o servidor X via MobaXterm. Mesmo 
 corrigindo a variável DISPLAY para assegurar a comunicação com o servidor X. 
 Os aplicativos gráficos como o xeyes e clock funcionam normalmente, mas quando
 rodo a cub3D dá um erro relacionado a extensão MIT-SHM e à sincronização de 
 múltiplas threads. Pedi ajuda para resolver.

Por fim, pedi ao meu amigo para rodar o código na máquina dele e o erro que 
voltou foi outro:

'double free ou corruption (out)'
'Aborted'

O que sinalizou problemas com o código e não necessariamente na interação do
servidor X do MobaXterm com o ambiente WSL2. 

A mensagem de erro "double free or corruption (out)" indica que há um problema 
no gerenciamento de memória do programa. Acontece geralmente quando o código 
tenta liberar a mesma área de memória mais de uma vez ou se há uma corrupção 
na estrutura de dados que gerencia a alocação de memória.

Para resolver, rodei o Valgrind, ajudando a detectar onde estavam ocorrendo 
desalocações duplas e outras questões de memória.
